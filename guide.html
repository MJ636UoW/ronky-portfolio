1. Ecosystem Architecture & File Synchronization

The Ronky Edits website is built as a hybrid project to facilitate both fast public browsing (via Vercel) and dynamic administration (via local WordPress on XAMPP).

Environment Location / Link Technology Content Source
WordPress Admin http://localhost/wordpress/wp-admin/ PHP & MySQL (Local XAMPP database) Customizer settings & database posts
Vercel (Live Web) https://ronky-portfolio.vercel.app/ Static HTML5, CSS3, & Vanilla JS Built files pushed to GitHub remote
⚠️ The Sync Boundary

Because Vercel is a static web host, making changes in your local WordPress dashboard does not automatically update the Vercel site. To apply updates to Vercel, you must build the static files (`npm run build`) and push them to GitHub. Once you perform the final migration to a live WordPress host (like Hostinger), your live site will be fully dynamic and automatically update whenever you save changes in your WordPress dashboard.

2. Editing Elements via WordPress Customizer

The majority of the content, texts, contact details, and images on the website are editable without touching a single line of code by using the WordPress Customizer.

1
Access the Customizer Panel

Go to your WordPress Admin dashboard and navigate to Appearance > Customize.

2
Hero Section Settings

Under Hero Section Settings, you can change:

  • Hero Title: The primary slogan. You can use HTML tags like <br> and <span class="accent-text text-glow"> to create styled glowing text.
  • Hero Subtitle: The descriptive paragraph text.
  • Hero Image: Upload the large showcase editorial image.
3
About Section Settings

Under About Section Settings, you can modify:

  • About Title & Text: The copy explaining your creative story.
  • Profile Image: The photo displayed inside the glass profile card.
  • Stats Counters: Edit the target numbers for Projects Completed, Happy Clients %, Countries Served, and Years Experience. The counters automatically animate when scrolled into view.
4
Services Settings

Under Services Settings, you can customize the 4 services cards:

  • Service Title & Description: Edit titles (e.g. photography, videography) and descriptions.
  • Gallery Media: Enter a comma-separated list of image URLs (or video URLs) that will display inside the immersive lightboxes when visitors click "More Details".
5
Contact & Social Settings

Under Contact Settings, you can change:

  • Email Address: Updates the mailto link on the email card.
  • WhatsApp Number: Updates the click-to-chat API redirection link.
  • Instagram Handle: Updates the link to your profile.
  • Fluent Form ID: Connects your custom form shortcode.

3. Changing Themes, Colors, and Typography (Custom CSS)

The site's visual theme uses modern styling design tokens. To change the overall color scheme, typography, background grid, or hover glow effects, you edit the CSS variables.

A. Key Theme Files

  • WordPress Theme CSS: ronky-portfolio-theme/assets/css/theme-style.css (junctioned to XAMPP).
  • Static Site CSS: src/style.css (used for compiling Vercel build).

B. Changing the Accent Color (e.g. Acid Green to Red or Gold)

Locate the :root selector at the top of the CSS file. You can change the primary color scheme by editing these color hex values:

:root {

--bg-primary: #0A0A0A; /* Page background color */

--bg-secondary: #121212; /* Cards/modals background */

--accent: #B8FF00; /* Main brand neon green color */

--accent-glow: rgba(184, 255, 0, 0.4); /* Glow border effect */

--text-primary: #FFFFFF;

}

To change the theme to a premium Hot Gold, change --accent to #FFDF00 and --accent-glow to rgba(255, 223, 0, 0.4).

C. Changing Typography

By default, titles use Syne and body texts use Plus Jakarta Sans. To change them:

  1. Change the Google Font embed links in your HTML/PHP headers (line 8 in guide.html or functions.php).
  2. Update the font-family properties in the CSS file:
    • Headers: font-family: 'YourNewFont', sans-serif;
    • Body Text: font-family: 'YourNewFont2', sans-serif;

4. Managing Portfolio Works and Taxonomies (Categories)

Your portfolio works are managed dynamically using standard WordPress Posts.

1
Create a Portfolio Work

Navigate to Posts > Add New. Enter your work title (e.g. SKATE MOTION).

2
Assign Categories (Taxonomies)

On the right-hand panel, select the category that represents this work. The categories are automatically created by the theme on load:

  • Fashion: Displays under the FASHION filter on the homepage.
  • Editorial: Displays under the EDITORIAL filter on the homepage.
  • Video: Triggers a video-preview playback rather than an image lightbox.

Note: You can check multiple categories. Do not check/create an "ALL" category.

3
Set the Featured Image (Thumbnail)

In the right sidebar, click Set featured image and upload your thumbnail. If you leave this empty, the theme will automatically cycle through the 4 default mockup images to keep the layout looking complete.

4
Add Video Playback (Optional)

For video works, scroll to the bottom of the editor page to find the Custom Fields box. Add a custom field named video_url and paste your direct MP4 video link (e.g., from your Media Library). The grid card will then play the video automatically on hover.

5. Adding Dynamic Testimonials

Instead of using static text boxes, clients' reviews are managed dynamically using the custom Testimonials sidebar menu.

1
Go to the Testimonials Panel

Click on the Testimonials menu item (speech bubble icon) in your WordPress left sidebar.

2
Create a New Review

Click Add New Testimonial and configure these fields:

  • Post Title: Enter the client's name (e.g. Amara Vance).
  • Main Editor Content Box: Type the client's review text.
  • Featured Image: Upload the client's profile picture or logo avatar.
  • Client Details Box (Meta Box): Enter their job description or brand (e.g. Founder, NEO-GRID Tokyo).
3
Publish and Auto-Scale

Click Publish. The reviews slider on the homepage will immediately extend itself. If you delete all testimonials, the site gracefully falls back to displaying the customizer-configured testimonials.

6. Customizing and Connecting Fluent Forms

The contact form is built on Fluent Forms, integrated with custom AJAX scripts to maintain a sleek design.

1
Locate and Customize the Form fields

In your WordPress left sidebar, click on Fluent Forms. You will see a form pre-created by the theme named Ronky Edits Portfolio Form. Click Edit.

You can add, edit, or rename fields. Ensure that the field name attributes match the form labels.

2
Check the Form ID

Note down the ID of the form from the list (e.g., [fluentform id="1"] means your ID is 1).

3
Connect the Form ID in WordPress Customizer

Go to **Appearance > Customize > Contact Settings**. Under the **Fluent Form ID** text field, input your form ID (e.g., 1) and click **Publish**. This connects the homepage form fields directly to your Fluent Forms database entries.

🔒 Submission Self-Healing Engine

The theme features a built-in self-healing algorithm inside `functions.php`. If you configure a custom Fluent Form that has different field keys, the theme automatically runs name matching to capture the submission and map it correctly without returning validation errors to the visitors.

7. Working with Elementor Page Builder

You can add Elementor templates or edit your custom layout structures.

1
Enable Elementor Support

Go to **Pages > Add New**, name your page, and click **Edit with Elementor**.

2
The Auto-Populating Template

The theme has an auto-population hook. When you create a new empty page in Elementor, it automatically loads our custom structured JSON layout template (`ronky-homepage-template.json`) containing our custom widgets. This lets you construct new custom layouts in seconds.

3
Managing Page Templates

If you build a page with Elementor and want it to display standard Elementor widgets, ensure the page template is set to **Elementor Full Width** in the page attributes sidebar.

8. Step-by-Step Live Hosting & Migration Guide

When you are ready to migrate the site from your local XAMPP machine to a live host (such as Hostinger, Bluehost, or GoDaddy), follow these steps:

Part A: Migrating your WordPress Site (to Live PHP/MySQL Hosting)

1
Install Migration Plugin Locally

On your local site (http://localhost/wordpress/wp-admin/), go to Plugins > Add New. Search for All-in-One WP Migration. Install and activate it.

2
Export the Local Database and Files

Navigate to All-in-One WP Migration > Export. Select Export To > File. Wait for the plugin to pack your database, themes, plugins, and media. Download the resulting .wpress file to your computer.

3
Set Up WordPress on Live Hosting

Log in to your hosting panel (e.g. Hostinger hPanel). Use the **Auto Installer** to install a fresh copy of WordPress on your domain (e.g. yourdomain.com).

4
Import your Local Site Live

Log in to your new live site (yourdomain.com/wp-admin/). Go to Plugins, install and activate All-in-One WP Migration. Go to Import > Import From > File, select the downloaded .wpress file, and click upload. Once completed, your live site will have all your portfolio works, customizer layouts, form configurations, and active themes restored!

Part B: Deploying your Static Web Site to Vercel (For static representation)

If you want to update your static showcase on Vercel:

  1. Make your changes to the static code files under the sandy/ folder.
  2. Open your terminal in the project root: C:\Users\manda\.gemini\antigravity\scratch\sandy\
  3. Compile the build: npm.cmd run build
  4. Commit the changes:
    • git add .
    • git commit -m "Updated theme designs"
  5. Push to GitHub: git push origin main
  6. Vercel detects the push and deploys the new static site automatically.
