RONKY EDITS

Complete User Guide & Technical Documentation

A comprehensive guide on managing, editing, and extending the Ronky Edits Gen-Z Portfolio Website. Learn how to configure dynamic testimonials, manage the auto-creating gallery archive, and deploy updates to both local environments (XAMPP) and live servers (Vercel).

Version 2.0 • June 2026

1. Understanding the Architecture

The Ronky Edits ecosystem consists of two separate structures to facilitate both dynamic backend management and fast static web serving:

Feature Local WordPress (XAMPP) Live Site (Vercel)
Purpose Dynamic database administration & local testing. Fast, globally distributed public showcase.
Technology PHP & MySQL (Local Database). Static HTML5, Vanilla CSS, and JavaScript.
Link / URL http://localhost/wordpress/ https://ronky-portfolio.vercel.app/
How Data Updates Saved inside your local database via Dashboard. Code changes pushed to GitHub remote.
⚠️ Important Sync Rule

Modifications made inside the local WordPress Customizer or Dashboard update your local machine's MySQL database. Because Vercel is a static host and has no database connection, local WordPress changes will not automatically push to Vercel. To update Vercel, you must edit the code files directly and push to GitHub, or migrate the theme to a dynamic WordPress host (like Hostinger).

2. Dynamic Testimonials CPT

Testimonials are fully dynamic. You can add, edit, or delete an unlimited number of testimonials directly from your WordPress sidebar, replacing the old 3-slot Customizer limit.

1
Access the Testimonials Menu

Log in to your WordPress dashboard (http://localhost/wordpress/wp-admin/). In the left sidebar, click on the new Testimonials menu item (distinguished by a speech bubble icon).

2
Add a New Review

Click on Add New Testimonial. Fill in the following fields:

  • Title: Enter the client's name (e.g. Sofia Laurent).
  • Content Box: Enter their review text/quote.
  • Featured Image (right panel): Upload the client's avatar image.
  • Client Details (bottom box): Enter their role/designation (e.g. Creative Lead, MAISON Agency).
3
Publish and Render

Click **Publish**. The review slider on the homepage will automatically expand and query your new testimonial. If no testimonial posts are created yet, the site gracefully falls back to displaying the three customizer-configured reviews.

3. Gallery Page & Conditional "Explore More" Button

The Gallery Page operates dynamically to display your full archive, while the homepage adapts based on how many works are added.

A. The Gallery Page

  • Automatic Page Creation: When you load the site, a page named "Portfolio Gallery" is automatically created in the database under the slug /gallery/.
  • Template Assignment: The theme automatically maps the custom page-gallery.php template to it. This template renders a full-page grid of all works, category filters, lightboxes, and a "Back to Home" button.
  • Customizer Logo Sync: The gallery header logo dynamically adapts to the brand name configured in the WordPress settings.

B. Conditional "Explore More" Button

The homepage limits the portfolio works to a maximum of 6 items to maintain visual balance and layout spacing.

  • Fewer than 6 items: If you have 6 or fewer published posts, all items will display on the homepage, and no CTA button will appear.
  • More than 6 items: If you publish a 7th post, the theme will automatically slice the display to show the latest 6, and **instantly auto-create the "Explore More" button** underneath the grid.
  • Connection: The button links dynamically to the Gallery Page (http://localhost/wordpress/gallery/), allowing visitors to browse the full archive.

4. How to Update the Live Vercel Site

To synchronize updates with Vercel, use Git.

1
Open Terminal / PowerShell

Navigate to the project root directory: C:\Users\manda\.gemini\antigravity\scratch\sandy\

2
Build Static Assets

Compile the code to prepare multi-page bundle paths (index.html and gallery.html):

npm.cmd run build

3
Stage and Commit Files

Stage the changes and commit them to Git:

git add .

git commit -m "Your description of updates"

4
Push to GitHub

Pushing to GitHub automatically triggers a clean rebuild on Vercel:

git push origin main
